New NGDP numbers are out today for the US, so that means I have to check several forecasts for accuracy. I would like to lead with a model that I seem to have forgotten to update all year: [dynamic equilibrium for the ratio of nominal output to total employed](http://informationtransfereconomics.blogspot.com/2017/03/the-quantity-theory-of-labor-and.html) (i.e. nominal output per employed, as I write _N/L_):



![](<media/dynamic equilibrium -gdp- data updates 1.png>)![](<media/dynamic equilibrium -gdp- data updates 2.png>)

This one is particularly good because the forecast was made near what the model saw as a turnaround point in _N/L_ (similar to the case of Bitcoin below, also forecast near a turnaround point) saying we should expect a return towards the trend growth rate of _N/L_ of 3.8% per annum. This return appears to be on track.



The [forecast of NGDP using the information equilibrium (IE) monetary model](http://informationtransfereconomics.blogspot.com/2017/04/update-to-predicted-path-of-ngdp.html) (i.e. a single factor of production where money — in this case physical currency — is that factor of production) is also "on track":



![](<media/ite NGDP path new data for Q2 2017 2.png>)![](<media/ite NGDP path new data for Q2 2017.png>)

The interesting part of this forecast is that the log-linear models are basically rejected.



In addition to NGDP, quarterly \[core\] PCE inflation was updated today. The NY Fed DSGE model forecast (as well as FOMC forecast) was for this data, and it's starting to do worse [compared to the IE monetary model](http://informationtransfereconomics.blogspot.com/2017/01/an-inflation-forecast-comparison-update.html) (now updated with monthly core PCE number as well):



![](<media/basic us price level m0 interst rates -compare FRB-US model- smoothed -main- 1.png>)![](<media/basic us price level m0 interst rates -compare FRB-US model- smoothed -main- 2 -1-.png>)
